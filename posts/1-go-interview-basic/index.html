Go 的值类型和引用类型有哪些？

  • 值类型 (Value Types): 创建该值的一个完整副本

    • 基本类型: int,float,bool,rune,string
    • 复合类型: array,struct

  • 引用类型 (Reference Types): 指向底层数据结构的指针

    • slice,map,channel,interface,function,pointer

slice

  • Go Slice 源码

    type slice struct {
    array unsafe.Pointer  // 指向底层数组的指针
    len   int             // 切片的长度（当前元素个数）
    cap   int             // 切片的容量（底层数组可容纳的最大元素个数）
}
  • Slice 扩容机制：当你 append 超过了 Cap 时，Go 会自动扩容
    • 旧版本（1.18 以前）：1024 字节以下翻倍，以上则增长 1.25 倍
    • 新版本（1.18 以后）：变得更加平滑，不再是以 1024 为分界点，而是通过一个公式计算，让扩容系数从 2 逐渐降到 1.25
    • 为什么要这么设计：因为如果一直翻倍，内存会浪费严重；如果一直加一点点，频繁申请内存又会拖慢你的代码运行速度。这其实是一种“时间”和“空间”的平衡术

  • 常见误区与注意事项

    • 共享底层数组：s1 := s[1:3] 和 s 共享同一底层数组，修改 s1[0] 会影响 s[1]
    • 扩容后独立：一旦 append 触发扩容，新 slice 与旧 slice 的底层数组分离
    • nil slice vs 空 slice：var s []int（nil，len=0, cap=0, array=nil）；make([]int, 0) 或 []int{}（非 nil，len=0, cap=0 或小容量）
    • 字符串与 Slice：string 底层类似（指针 + len），但不可变

Map

  • Go Map 源码
  • hash table 核心两个结构组成 hmap（哈希表头部）和 bmap （bucket 桶）
  • 在 Go 1.23+ 引入了新的 Swiss table 实现

  • Map 的底层：hmap 与 bmap

    • 它由一个 hmap 结构体控制，里面包含多个 buckets（桶）
    • 每个桶是一个 bmap，最多存 8 个键值对

    • 当哈希冲突发生时，Go 使用的是链地址法（通过 overflow 指针指向溢出桶）

    • hmap 大概伪代码

      type hmap struct {
    count     int          // map 中当前键值对的数量（len(m) 返回的值，必须放在第一个字段）
    flags     uint8        // 标记位（如是否正在扩容、是否被迭代等）
    B         uint8        // 桶的数量的对数：桶数 = 2^B （例如 B=3 时有 8 个桶）
    noverflow uint16       // 溢出桶（overflow bucket）的近似数量
    hash0     uint32       // 哈希种子（随机值），用于避免哈希碰撞攻击
    buckets   unsafe.Pointer // 指向桶数组的指针（正常桶数组，大小为 2^B 个桶）
    oldbuckets unsafe.Pointer // 扩容期间指向旧桶数组的指针（扩容时非 nil）
    nevacuate uintptr       // 扩容进度计数器（已搬迁的桶数量）
    // 还有 extra *mapextra 等字段，用于记录指针等额外信息
}

      type bmap struct {
    tophash [bucketCnt]uint8  // 每个 key 的 hash 值的高 8 位（tophash）
    // 后面紧跟着：8 个 key（连续存放）
    // 然后是：8 个 value（连续存放）
    // 最后是：overflow *bmap（指向溢出桶的指针）
}

  • Map 的扩容

    • 等量扩容（整理）： 键值对没满，但溢出桶太多了（通常是因为删除了很多数据），这时候会搬迁数据，让布局更紧凑
    • 翻倍扩容： 装载因子（Load Factor）超过 6.5（即：平均每个桶存了超过 6.5 个元素），这时会创建一个两倍大的新桶数组
    • 扩容渐进式：Go 的 Map 扩容是渐进式的。它不会一次性把所有数据搬完（那会卡死业务代码），而是在你后续进行 assign（赋值）或 delete（删除）操作时，顺便搬迁一点点
