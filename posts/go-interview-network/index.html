网络专项

高频题（带答）

1. TCP 三次握手与四次挥手？

答： 三次握手建立连接（SYN/SYN-ACK/ACK），四次挥手断开（FIN/ACK/FIN/ACK），确保双方收发完成。

2. 为什么有 TIME_WAIT？

答： 确保最后 ACK 可达并让旧连接数据包过期，避免影响新连接。

3. HTTP 与 HTTPS 区别？

答： HTTPS 在 HTTP 之上增加 TLS 握手与加密，保证传输安全与身份校验。

4. HTTP/1.1 与 HTTP/2 区别？

答： HTTP/2 支持多路复用、头部压缩、二进制帧，减少连接开销。

5. REST 与 gRPC 的差异？

答： REST 多基于 HTTP/JSON，易调试；gRPC 基于 HTTP/2+Protobuf，性能更好、强类型。

6. 长连接与短连接怎么选？

答： 高频请求适合长连接降低握手开销；低频或无状态场景短连接足够。

7. 常见的负载均衡策略？

答： 轮询、加权轮询、最少连接、一致性哈希。

8. DNS 解析过程？

答： 客户端缓存 -> 本地 DNS -> 递归/迭代解析 -> 权威 DNS 返回记录。

9. TCP 粘包是什么？

答： TCP 是字节流协议，应用层需要自定义协议（长度前缀/分隔符）解决粘包。

10. Go 的 HTTP Client 如何设置超时？

答： 使用 http.Client{Timeout: ...} 或在 Transport 中设置连接、响应头等超时。

11. TCP 与 UDP 的区别？

答： TCP 面向连接、可靠传输；UDP 无连接、开销小但不保证可靠性。

12. Nagle 算法与延迟 ACK 有何影响？

答： 可能引发小包延迟，低延迟场景可关闭 Nagle（TCP_NODELAY）。

13. TLS 握手简述？

答： 协商版本与密码套件、交换密钥、验证证书，建立安全通道。

14. HTTP Keep-Alive 的作用？

答： 复用连接减少握手成本，提高性能。

15. 常见 HTTP 状态码分类？

答： 2xx 成功，3xx 重定向，4xx 客户端错误，5xx 服务端错误。

16. CORS 是什么？

答： 浏览器跨域策略，通过响应头控制跨域访问权限。

17. WebSocket 的特点？

答： 基于 HTTP 握手升级为全双工长连接，适合实时通讯。

18. CDN 的作用？

答： 就近分发内容，降低源站压力与延迟。

19. MTU 与分片问题？

答： 超过 MTU 的包会分片，分片丢失会导致整体丢失，影响性能。

20. QUIC 相对 TCP 的优势？

答： 基于 UDP，内建 TLS 1.3，避免队头阻塞，多路复用更好。

21. gRPC 支持的四种流式模式？

答： Unary、Server streaming、Client streaming、Bidirectional streaming。

22. 正向代理与反向代理区别？

答： 正向代理代表客户端访问外网；反向代理代表服务端对外提供统一入口。

23. 幂等方法有哪些？

答： GET/PUT/DELETE/HEAD 通常幂等；POST 非幂等。

24. SYN Flood 的缓解方式？

答： SYN cookies、调大 backlog、限流与防火墙。

25. L4 与 L7 负载均衡区别？

答： L4 基于 TCP/UDP 转发；L7 能识别协议并按 URL/Header 路由。

26. 连接池如何提升性能？

答： 复用 TCP 连接，减少握手与 TLS 成本，提高吞吐。

27. 什么是半连接队列与全连接队列？

答： 半连接存放 SYN 未完成握手；全连接存放已完成握手等待 accept。

28. TCP 为什么会队头阻塞？

答： TCP 保证顺序，丢包导致后续包等待重传。

29. 如何设计重试策略？

答： 设置超时、限制重试次数、指数退避，并确保幂等。

30. Go 的 Transport 常用参数？

答： MaxIdleConns、MaxIdleConnsPerHost、IdleConnTimeout、TLSHandshakeTimeout。

深入追问（10题）

31. TCP 三次握手的目的？

答： 建立双方收发能力与初始序列号同步，避免历史连接干扰。

32. 为什么会有 TIME_WAIT？

答： 让被动关闭方能收到最后 ACK，确保旧连接的延迟包不会污染新连接。

33. HTTP Keep-Alive 的收益与风险？

答： 减少连接建立开销，但需要连接池与空闲回收，避免连接泄漏。

34. HTTP/2 的多路复用解决了什么问题？

答： 同一连接上并发多请求，避免队头阻塞，提升并发与首包时间。

35. TLS 握手涉及哪些关键步骤？

答： 协商版本与套件、证书验证、密钥交换、生成会话密钥。

36. DNS 缓存的层次有哪些？

答： 浏览器/系统缓存、本地递归解析器缓存、权威服务器 TTL。

37. 四层与七层负载均衡的差异？

答： 四层按连接转发（性能高），七层基于请求内容（功能强）。

38. 重试策略为什么要配合幂等性？

答： 非幂等请求重试可能导致副作用放大，需要幂等设计或请求去重。

39. WebSocket 与 SSE 的选择？

答： WebSocket 双向通信；SSE 单向推送，基于 HTTP 更易穿透代理。

40. QUIC 相比 TCP 的优势？

答： 基于 UDP 实现多路复用与 0-RTT，减少握手延迟并避免队头阻塞。

追问与陷阱（10题）

41. TCP 为什么需要四次挥手？

答： 双向关闭需要各自确认，确保数据完整发送与接收。

42. HTTPS 一定比 HTTP 慢吗？

答： 握手有开销，但连接复用与硬件加速可降低影响；对长连接影响较小。

43. 头部压缩（HPACK）的意义？

答： 减少重复头部开销，提升带宽利用率与延迟表现。

44. 反向代理为什么能隐藏源站？

答： 客户端只与代理交互，源站不暴露于公网。

45. TCP 拥塞控制的目标？

答： 在保证公平与稳定的前提下最大化吞吐。

46. 为什么需要应用层心跳？

答： 及时检测连接是否可用，避免半开连接长期占用资源。

47. HTTP 连接池常见问题？

答： 连接泄漏、空闲过多、慢连接占用，需要合理超时与上限。

48. TCP_NODELAY 的作用？

答： 关闭 Nagle 算法，降低小包延迟但可能增加带宽消耗。

49. DNS 解析为什么可能变慢？

答： 缓存失效、递归解析链路长、权威服务器延迟。

50. 代理转发导致的客户端 IP 丢失如何处理？

答： 使用 X-Forwarded-For 或 X-Real-IP 传递真实 IP。

package main

import (
    "fmt"
    "io"
    "net/http"
    "net/http/httptest"
)

func main() {
    ts := httptest.NewServer(http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprint(w, "ok")
    }))
    defer ts.Close()

    resp, _ := http.Get(ts.URL)
    defer resp.Body.Close()
    body, _ := io.ReadAll(resp.Body)

    fmt.Println(resp.StatusCode)
    fmt.Println(string(body))
}
