Go 标准库专题

Posted Apr 8, 2026
By Cc 5 min read

Go 标准库专题

高频题（带答）

1. net/http 中 Client 是否可复用？

答： 可以且应该复用；每次新建会导致连接无法复用与资源浪费。

2. http.Transport 的作用？

答： 管理连接池、代理、TLS、Keep-Alive 等底层传输细节。

3. http.Server 的几个关键超时参数？

答： ReadTimeout、WriteTimeout、IdleTimeout、ReadHeaderTimeout。

4. context 在标准库中的常见使用点？

答： http、database/sql、os/exec、net 等均支持取消与超时。

5. time.After 与 time.NewTimer 的差异？

答： time.After 每次创建新 Timer；高频场景应复用 time.NewTimer。

6. time.Ticker 的正确释放方式？

答： 使用 Stop() 释放资源，避免泄漏。

7. sync.Pool 的适用场景？

答： 高并发下复用临时对象，降低 GC 压力，但不保证命中。

8. sync.Once 的典型场景？

答： 单例初始化、配置加载、懒加载资源。

9. sync.Cond 用于什么？

答： 多 goroutine 等待某条件满足时统一唤醒。

10. sync.Map 适合哪些场景？

答： 读多写少、key 稳定，或并发访问且无法加锁的场景。

11. bytes.Buffer 与 strings.Builder 的区别？

答： strings.Builder 专用于构造字符串，避免额外拷贝；bytes.Buffer 更通用。

12. bufio.Reader 的价值？

答： 减少系统调用次数，提高 IO 效率。

13. io.Reader 与 io.Writer 的设计哲学？

答： 以流式抽象统一处理数据源与数据目标，易于组合与复用。

14. io.Copy 的典型用途？

答： 在两个流之间高效复制数据，常用于转发与下载。

15. os.File 与 bufio 如何配合？

答： os.File 提供原始 IO，bufio 提供缓冲提升性能。

16. encoding/json 中 omitempty 的行为？

答： 字段为零值时不输出；指针为 nil 时不输出。

17. json.RawMessage 的用途？

答： 延迟解析或保留原始 JSON 片段。

18. database/sql 的连接池如何调优？

答： 设置 SetMaxOpenConns、SetMaxIdleConns、SetConnMaxLifetime。

19. database/sql 的 Prepare 是否一定更快？

答： 不一定；短 SQL 可能成本更高，应按场景判断。

20. net 包里的 Dialer 常用于什么？

答： 设置连接超时、保活参数、控制连接行为。

21. os/exec 如何设置超时？

答： 使用 exec.CommandContext 结合 context.WithTimeout。

22. errors.Is 与 errors.As 的区别？

答： Is 用于判断是否匹配目标错误；As 用于获取指定类型错误。

23. log 与 log/slog 的差异？

答： log 简单输出；slog 支持结构化日志与上下文字段。

24. regexp 的性能注意点？

答： 正则需预编译，避免每次调用都重新解析。

25. context.Value 的正确使用？

答： 仅传递请求范围元数据，不用于业务参数。

26. net/http 的 RoundTripper 适用场景？

答： 自定义请求发送逻辑，如打点、重试、注入 Header。

27. crypto/rand 与 math/rand 的区别？

答： 前者安全随机，后者伪随机，安全场景必须用 crypto/rand。

28. sort.Slice 的使用注意点？

答： 比较函数必须稳定、无副作用，否则排序结果不可靠。

29. strconv 常见坑？

答： 进制参数设置错误、忽略错误返回导致数据异常。

30. 标准库中如何做 base64 编解码？

答： 使用 encoding/base64 的 StdEncoding。

package main

import (
    "fmt"
    "net/http"
    "net/http/httptest"
)

func main() {
    ts := httptest.NewServer(http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprint(w, "hello")
    }))
    defer ts.Close()

    resp, _ := http.Get(ts.URL)
    defer resp.Body.Close()

    fmt.Println(resp.StatusCode)
}
