常见面试题（带答）

1. GMP 模型是什么？

答： G 是 goroutine，M 是 OS 线程，P 是处理器（运行队列）。P 负责调度 G 在 M 上执行。

2. Go 的 GC 过程？

答： Go 使用三色标记清扫，采用并发 GC，主要阶段包括标记开始、并发标记、标记终止、清扫。

3. pprof 常用入口与命令？

答： net/http/pprof 提供 /debug/pprof/，常用 go tool pprof 或 go tool pprof -http 查看。

4. sync.Pool 的作用？

答： 用于对象复用，降低频繁分配与 GC 压力，适合临时对象。

5. atomic 与锁的选择？

答： 原子操作适合简单数值和指针；锁适合复杂临界区。原子更轻量但受限。

6. 什么时候会出现内存对齐问题？

答： 结构体字段顺序不合理导致 padding 增多。可通过调整字段顺序降低内存占用。

7. go mod 的依赖治理实践？

答： 使用 go mod tidy 清理依赖；用 replace 管理本地依赖；避免 go get -u 无脑升级。

8. GC 调优常见手段？

答： 减少短生命周期对象、复用 buffer、适当调整 GOGC、减少大对象分配。

9. 如何定位 CPU/内存热点？

答： 使用 pprof 的 cpu、heap、allocs，结合 top/list/web 分析热点函数。

10. 大量 goroutine 阻塞如何排查？

答： 查看 /debug/pprof/goroutine 或 runtime/pprof，定位阻塞点和锁竞争。

11. GC 的 STW 发生在哪些阶段？

答： 标记开始与标记终止会有短暂 STW；并发标记与清扫是并发进行。

12. GOGC 的含义与调优思路？

答： 表示堆增长比例阈值（默认 100）；调大可减少 GC 频率但增加内存占用，调小相反。

13. 如何观察 GC 压力？

答： 通过 GODEBUG=gctrace=1 或 pprof 的 heap/allocs 观察分配与回收趋势。

14. 逃逸分析对性能的影响？

答： 逃逸到堆会增加 GC 压力和分配开销；可通过减少指针返回、避免闭包捕获降低逃逸。

15. 结构体字段顺序如何影响内存？

答： 不同类型对齐要求不同，合理排序字段可减少 padding，降低内存占用。

16. pprof 中 heap 与 allocs 有何区别？

答： heap 看当前存活对象，allocs 看累计分配，排查内存泄漏时优先 heap。

17. 如何定位锁竞争？

答： 打开 mutex profile（runtime.SetMutexProfileFraction），配合 go tool pprof 查看热点。

18. 如何定位阻塞与调度问题？

答： 使用 block profile 或 go tool trace 查看 goroutine 阻塞原因与调度链路。

19. 调度器抢占机制的变化？

答： Go 1.14+ 支持异步抢占，减少“长时间独占 CPU”的 goroutine。

20. GOMAXPROCS 在 CPU 密集型服务中的实践？

答： 通常设为 CPU 核数；容器环境需关注 CPU quota，否则容易过度并行。

21. 如何系统性降低分配？

答： 复用 buffer、预分配 slice、避免 fmt.Sprintf 临时对象、减少 interface 装箱。

22. sync.Pool 的坑有哪些？

答： GC 可能清空 pool；对象需要重置；不适合“必须存在”的缓存。

23. map 扩容的表现是什么？

答： 扩容时桶会渐进式搬迁（evacuate），可能导致写入延迟抖动。

24. channel 的实现要点？

答： 包含环形缓冲区、发送/接收等待队列和锁；无缓冲 channel 会直接同步发送与接收。

25. net/http 服务端的性能参数有哪些？

答： ReadTimeout/WriteTimeout/IdleTimeout 控制慢请求；MaxHeaderBytes 限制头部大小。

26. 如何分析 goroutine 泄漏？

答： 采集 goroutine profile，查看阻塞点；结合 context 与退出条件排查。

27. go mod 的主版本号规则？

答： v2+ 需要在 module path 中体现（如 module example.com/x/v2）。

28. go mod vendor 的适用场景？

答： 构建环境不可信或离线时；也可用于依赖审计。

29. pprof 的 cpu 采样注意点？

答： 采样窗口需足够长且有负载；短窗口容易误判热点。

30. 生产环境如何“安全”开启 profiling？

答： 只对内网开放 /debug/pprof，或加鉴权/反代白名单，避免泄露敏感信息。
