文件类型

  • 普通文件 -

  • 块设备文件 d/c

  • 字符设备文件 d/c
  • socket 文件 s

文件属性权限

  • 读…………………………r
  • 写…………………………w
  • 可执行…………………X
示例
1
2
3
4
5

-rwxrw-r--
[-]表示普通文件
[rwx]表示读写执行权限
[rw-]所属与同一组的用户就有读写权限
[r--]表示其他用户对该文件只有读权限

-rwxrwxrx：为文件最大权限，对应的编码为 777，表示任何用户都有读写执行此文件

改变文件所有权

  • chown
  • chgrp

改变文件权限

chmod
  • r—-4—–读———read
  • w—2—–写——–write
  • x—-1—–执行—–[execute]

 -n, --no-mtab           不写 /etc/mtab
 -o, --options <列表>    挂载选项列表，以英文逗号分隔
 -O, --test-opts <列表>  限制文件系统集合(和 -a 选项一起使用)

 -V, --version           显示版本信息并退出

 -h, --help     显示此帮助并退出
 -V, --version  输出版本信息并退出

源：
 -L, --label <标签>      同 LABEL=<label>

交换空间管理的基本操作

交换空间（Swap Space）是 Linux 系统中用于临时存储数据的一种内存管理机制。当物理内存（RAM）不足时，系统会将不活跃的数据转移到交换空间，以释放 RAM 供活跃进程使用。

1. 查看当前交换空间使用情况

使用以下命令查看当前的交换空间状态：

1

swapon --show

或者：

1

free -h

2. 创建交换文件

如果你需要增加交换空间，可以创建一个交换文件。例如，创建一个 2GB 的交换文件：

1

sudo fallocate -l 2G /swapfile

如果 fallocate 不可用，可以使用 dd 命令：

1

sudo dd if=/dev/zero of=/swapfile bs=1G count=2

3. 设置交换文件权限

为了安全起见，设置交换文件的权限：

1

sudo chmod 600 /swapfile

4. 将文件设置为交换空间

使用以下命令将文件设置为交换空间：

1

sudo mkswap /swapfile

5. 启用交换空间

启用交换空间：

1

sudo swapon /swapfile

6. 确认交换空间已启用

再次使用以下命令查看交换空间状态，确认其已启用：

1

swapon --show

7. 使交换空间永久生效

要使交换空间在重启后仍然有效，需要将其添加到 /etc/fstab 文件中。打开 /etc/fstab 文件：

1

sudo nano /etc/fstab

添加以下行：

1

/swapfile none swap sw 0 0

保存并退出。

8. 调整交换使用优先级

你可以通过调整交换优先级来控制内存的使用。swappiness 参数控制系统使用交换空间的倾向，范围从 0 到 100。值越低，系统越倾向于使用物理内存。

查看当前的 swappiness 值：

1

cat /proc/sys/vm/swappiness

设置 swappiness 值为 10（例如）：

1

sudo sysctl vm.swappiness=10

要使该设置在重启后仍然有效，可以将其添加到 /etc/sysctl.conf 文件中：

1

echo "vm.swappiness=10" | sudo tee -a /etc/sysctl.conf

9. 禁用交换空间

如果要禁用交换空间，可以使用以下命令：

1

sudo swapoff /swapfile

注意事项

  • 性能影响：虽然交换空间可以防止系统崩溃，但使用交换空间的速度比物理内存慢，因此最好确保有足够的 RAM。
  • 监控使用情况：定期监控交换空间的使用情况，以便在必要时进行调整。
