缓存一致性专题

高频题（带答）

1. 什么是缓存一致性问题？

答： 缓存与数据库的数据短暂不一致，导致读到旧数据或脏数据。

2. Cache Aside 的标准写流程？

答： 先写数据库，再删除缓存；必要时延迟再删以兜底并发脏读。

3. Cache Aside 的读流程？

答： 先读缓存，未命中再读数据库并回填缓存。

4. Write-Through 的特点？

答： 写缓存即写库，保证一致性但写延迟更高。

5. Write-Back 的风险？

答： 写入先落缓存，异步刷库，吞吐高但存在丢数据风险。

6. Read-Through 与 Cache Aside 的差别？

答： Read-Through 由缓存层负责回源与填充；Cache Aside 由业务层控制。

7. “双删”策略是什么？

答： 写库前/后各删一次缓存，降低并发读写造成的脏缓存概率。

8. 延迟双删的作用？

答： 写库后延迟删除，覆盖并发回填导致的旧值问题。

9. 如何用 CDC 提升一致性？

答： 订阅 binlog/CDC 变更流，异步刷新或失效缓存。

10. 缓存穿透的解决方案？

答： 布隆过滤器、缓存空值、参数校验。

11. 缓存击穿的解决方案？

答： 互斥锁/单飞、逻辑过期、热点预热。

12. 缓存雪崩的解决方案？

答： 过期时间随机化、分批失效、限流与降级。

13. 逻辑过期的使用场景？

答： 热点数据允许短暂旧值，后台异步刷新，保证高可用。

14. 互斥锁回源的注意点？

答： 设置锁超时与重试，避免锁死和长时间阻塞。

15. 负缓存（缓存空值）有什么风险？

答： 可能掩盖后续真实数据写入，需设置短 TTL。

16. 热点 key 的治理手段？

答： 本地缓存、分片、读写分离、限流。

17. big key 的影响？

答： 阻塞主线程、影响响应延迟；应拆分或避免超大 value。

18. 缓存预热的价值？

答： 避免上线后冷启动击穿，提升首批请求的稳定性。

19. 多级缓存一致性怎么保证？

答： 通过统一失效通知或版本号控制，避免各级缓存不一致。

20. 本地缓存失效方案？

答： 设置 TTL + 订阅变更通知，或定期校验版本号。

21. 缓存 key 设计要注意什么？

答： 具备业务维度、避免过长、保持可读且可扩展。

22. 缓存淘汰策略与一致性的关系？

答： 淘汰会导致回源，需结合热点保护与限流避免穿透。

23. 数据版本号如何用于一致性？

答： 写库时生成版本号，缓存带版本，回源时比对避免旧值覆盖。

24. CAS 更新缓存的意义？

答： 比对版本或时间戳，避免并发写入覆盖新值。

25. 事务与缓存如何协同？

答： 在事务提交后再删除/更新缓存，避免回滚造成脏数据。

26. 延迟队列在一致性中的作用？

答： 用于延迟双删或异步刷新，降低并发读写导致的脏读。

27. Stale-While-Revalidate 的思路？

答： 先返回旧值，后台刷新，降低用户感知延迟。

28. Refresh-Ahead 的思路？

答： 在过期前提前刷新，减少过期瞬间的回源洪峰。

29. 缓存一致性的监控指标？

答： 命中率、回源率、热点 key、过期分布、回源耗时。

30. 一致性策略选择的关键因素？

答： 业务读写比例、可接受的延迟与数据新鲜度要求。

追问与陷阱（10题）

31. “先删缓存再写库”最危险的场景？

答： 并发回源写入旧值，导致缓存被旧数据覆盖。

32. 延迟双删的延迟时间如何选？

答： 依据平均回源时间与写入耗时，通常略大于回源链路耗时。

33. 缓存穿透为什么会拖垮数据库？

答： 大量请求绕过缓存直接打到 DB，造成负载飙升。

34. 布隆过滤器的误判风险？

答： 可能将不存在的 key 误判为存在，导致少量回源；不会漏判。

35. 多级缓存回源顺序如何设计？

答： 先本地再分布式，再数据库，逐级回源。

36. 为什么要加随机过期？

答： 避免大量 key 同时过期导致雪崩。

37. 缓存一致性监控看什么？

答： 命中率、回源率、回源耗时、热点 key、异常峰值。

38. 为什么“空值缓存”需要短 TTL？

答： 防止真实数据写入后仍被空值挡住。

39. 双写一致性如何确保？

答： 以数据库为准，通过变更流或补偿任务修正缓存。

40. 缓存更新是删还是更新？

答： 删更安全，更新更快但容易写旧值，需版本控制。

package main

import "fmt"

func main() {
    db := map[string]string{"k": "v1"}
    cache := map[string]string{"k": "old"}

    // 写库后删缓存
    db["k"] = "v2"
    delete(cache, "k")

    // 读流程：先读缓存，未命中回源并回填
    if v, ok := cache["k"]; ok {
        fmt.Println("cache", v)
    } else {
        v := db["k"]
        cache["k"] = v
        fmt.Println("db", v)
    }
}
